Typography System

A clear, readable typography hierarchy optimized for financial data and commercial banking workflows.

Font Stack

/* Primary Font Stack */ font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif; /* Monospace for Data */ font-family: 'Courier New', 'Monaco', monospace;

Type Scale & Examples

H1 - Page Titles (32px, 600 weight)

H2 - Section Headers (24px, 600 weight)

H3 - Card Titles (20px, 500 weight)

Body Text - Main content (16px, 400 weight) - Optimized for reading financial information and detailed banking workflows with sufficient line height for accessibility.

Secondary Text - Supporting information (14px, 400 weight) - Used for metadata, descriptions, and supplementary content.

Small Text - Labels and captions (12px, 400 weight) - Minimum size for accessibility compliance.

Layout Principles

  • 4px Base Unit: All spacing derives from 4px for perfect pixel alignment
  • Responsive Grid: 12-column system adapts to mobile, tablet, and desktop
  • Content Width: Maximum 1200px prevents excessive line lengths
  • Touch Targets: Minimum 44px for mobile accessibility
  • Breathing Room: Adequate whitespace reduces cognitive load

Responsive Breakpoints

/* Responsive Breakpoints */ --breakpoint-mobile: 320px; /* Small phones */ --breakpoint-tablet: 768px; /* Tablets */ --breakpoint-desktop: 1024px; /* Desktop */ --breakpoint-large: 1440px; /* Large screens */

Accessibility Standards

Comprehensive accessibility guidelines ensuring the design system meets WCAG 2.1 AA standards and provides inclusive experiences for all users.

Color & Contrast

Color Combination Contrast Ratio WCAG Level Use Case
Text Primary on White 13.1:1 AAA Body text, headings
Text Secondary on White 7.2:1 AAA Supporting text
Truist Purple on White 6.8:1 AA Links, interactive elements
White on Truist Purple 6.8:1 AA Primary buttons, badges

Keyboard Navigation

  • Focus Indicators: Visible 2px outline for all interactive elements
  • Tab Order: Logical sequence following visual hierarchy
  • Skip Links: Allow users to bypass navigation
  • Escape Functionality: Close modals and overlays with Escape key

Screen Reader Support

  • Semantic HTML: Proper heading hierarchy and landmark elements
  • ARIA Labels: Descriptive labels for complex interactions
  • Status Updates: Live regions announce dynamic content changes
  • Form Labels: Clear association between labels and form controls
